# Настройка личных номеров WhatsApp для складов

## Описание

Теперь можно привязать личный номер WhatsApp к каждому складу. Заявки будут автоматически отправляться на личный номер ответственного за склад.

## Приоритет отправки заявок

Система использует следующий приоритет при отправке заявок в WhatsApp:

```
1️⃣ Личный номер склада (если настроен)
   ↓
2️⃣ Группа склада (если настроена)
   ↓
3️⃣ Общая группа WhatsApp (из .env)
   ↓
4️⃣ Общий личный получатель (из .env)
```

## Установка

### 1. Добавьте колонку в базу данных:

```bash
node telegram-order-bot/add-whatsapp-phone-column.js
```

Эта команда добавит колонку `whatsapp_phone` в таблицу `warehouses`.

### 2. Перезапустите бота:

```bash
npm start
```

## Команды для настройки

### Привязать личный номер к складу:

```
/setwhatsappphone Название_склада | Номер_телефона
```

**Примеры:**
```
/setwhatsappphone Склад №1 | 992900000000
/setwhatsappphone Главный склад | 992901234567
/setwhatsappphone Центральный | 992909999999
```

**Важно:**
- Номер указывается без `+` и пробелов
- Только цифры
- Формат: код страны + номер (например: 992900000000)

### Отвязать личный номер от склада:

```
/removewhatsappphone Название_склада
```

**Пример:**
```
/removewhatsappphone Склад №1
```

### Посмотреть все склады с настройками:

```
/checkwarehouses
```

или через кнопку **"📋 Список складов"** в админ-панели.

## Примеры использования

### Пример 1: Склад с личным номером

```bash
# Добавляем склад
/addwarehouse2 Склад №1

# Привязываем личный номер
/setwhatsappphone Склад №1 | 992900000000
```

**Результат:** Все заявки для "Склад №1" будут отправляться на номер +992900000000

### Пример 2: Склад с группой и личным номером

```bash
# Добавляем склад
/addwarehouse2 Главный склад

# Привязываем группу
/setwhatsapp Главный склад | 120363XXXXXXXXXX@g.us

# Привязываем личный номер (приоритет выше!)
/setwhatsappphone Главный склад | 992901234567
```

**Результат:** Заявки будут отправляться на личный номер +992901234567 (группа игнорируется)

### Пример 3: Переключение с группы на личный номер

```bash
# Сначала был настроен на группу
/setwhatsapp Склад №2 | 120363XXXXXXXXXX@g.us

# Теперь переключаем на личный номер
/setwhatsappphone Склад №2 | 992909999999
```

**Результат:** Заявки теперь идут на личный номер, группа остается в настройках как резерв

### Пример 4: Отключение личного номера

```bash
# Отвязываем личный номер
/removewhatsappphone Склад №2
```

**Результат:** Заявки снова будут отправляться в группу (если настроена) или в общую группу

## Логика работы в коде

### Функция отправки (bot.js):

```javascript
// Получаем настройки склада
const settings = await database.getWarehouseWhatsAppSettings(data.warehouse);

// Приоритет 1: Личный номер
if (settings.whatsapp_phone) {
  whatsappSent = await whatsapp.sendMessage(orderMessage, settings.whatsapp_phone);
}
// Приоритет 2: Группа склада
else if (settings.whatsapp_group_id) {
  whatsappSent = await whatsapp.sendToGroup(orderMessage, settings.whatsapp_group_id);
}
// Приоритет 3: Общие настройки
else {
  // Используем WHATSAPP_GROUP_ID или WHATSAPP_RECIPIENT из .env
}
```

### Новые функции в database.js:

```javascript
// Обновить личный номер склада
await database.updateWarehouseWhatsAppPhone(warehouseName, phoneNumber);

// Получить личный номер склада
const phone = await database.getWarehouseWhatsAppPhone(warehouseName);

// Получить все настройки WhatsApp склада
const settings = await database.getWarehouseWhatsAppSettings(warehouseName);
// Возвращает: { whatsapp_group_id, whatsapp_phone }
```

### Обновленная функция sendMessage (whatsapp.js):

```javascript
// Теперь принимает номер как параметр
await whatsapp.sendMessage(message, phoneNumber);

// Если номер не указан, использует WHATSAPP_RECIPIENT из .env
await whatsapp.sendMessage(message);
```

## Структура базы данных

### Таблица warehouses:

```sql
CREATE TABLE warehouses (
  id SERIAL PRIMARY KEY,
  name TEXT NOT NULL UNIQUE,
  whatsapp_group_id TEXT,        -- ID группы WhatsApp
  whatsapp_phone TEXT,            -- Личный номер WhatsApp (НОВОЕ!)
  is_active INTEGER DEFAULT 1,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## Преимущества

✅ Каждый склад может иметь своего ответственного  
✅ Заявки приходят напрямую ответственному лицу  
✅ Не нужно создавать отдельные группы для каждого склада  
✅ Легко переключаться между личным номером и группой  
✅ Приоритетная система отправки (личный номер → группа → общие настройки)  
✅ Простая настройка через команды бота  
✅ Возможность комбинировать: одни склады на личные номера, другие в группы  

## Возможные проблемы

### ❌ "Колонка whatsapp_phone не существует"

**Решение:** Запустите миграцию:
```bash
node telegram-order-bot/add-whatsapp-phone-column.js
```

### ❌ "Неверный формат номера"

**Причина:** Номер содержит недопустимые символы  
**Решение:** Используйте только цифры без + и пробелов:
- ✅ Правильно: `992900000000`
- ❌ Неправильно: `+992 90 000 00 00`
- ❌ Неправильно: `+992900000000`

### ❌ Сообщение не приходит на личный номер

**Проверьте:**
1. Номер указан правильно (без + и пробелов)
2. Green-API настроен корректно
3. Номер добавлен в контакты WhatsApp
4. Проверьте логи бота на наличие ошибок

## Рекомендации

1. **Используйте личные номера** для небольших складов с одним ответственным
2. **Используйте группы** для крупных складов с несколькими сотрудниками
3. **Комбинируйте** - можно настроить и номер, и группу (приоритет у номера)
4. **Тестируйте** настройки после каждого изменения
5. **Документируйте** какой склад на какой номер настроен

## Миграция с групп на личные номера

Если у вас уже настроены группы и вы хотите перейти на личные номера:

```bash
# Шаг 1: Посмотрите текущие настройки
/checkwarehouses

# Шаг 2: Добавьте личные номера (группы останутся как резерв)
/setwhatsappphone Склад №1 | 992900000000
/setwhatsappphone Склад №2 | 992901234567

# Шаг 3: Проверьте настройки
/checkwarehouses

# Шаг 4: Создайте тестовую заявку для проверки
```

Группы останутся в настройках и будут использоваться, если вы отключите личные номера.
