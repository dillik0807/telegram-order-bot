# Выбор клиента при создании заявки администратором

## Описание изменений

Добавлена возможность для администратора выбирать клиента из списка зарегистрированных клиентов при создании заявки на поставку товаров. Список клиентов отображается в виде inline-кнопок прямо в сообщении.

## Как это работает

### Для администратора:

1. Нажмите кнопку **"📦 Создать заявку"**
2. Выберите склад
3. Добавьте товары в заявку
4. Нажмите **"✅ Продолжить"**
5. Система покажет список всех зарегистрированных клиентов в виде inline-кнопок в формате: `Имя (Телефон)`
6. Нажмите на кнопку с нужным клиентом
7. Система автоматически подставит имя и телефон выбранного клиента
8. Введите номер транспорта и комментарий
9. Подтвердите заявку

### Для обычных клиентов:

Процесс создания заявки остался без изменений - клиенты продолжают использовать кнопку **"🏬 Склад"** и заполняют заявку от своего имени.

## Порядок заполнения заявки администратором:

```
1. 📦 Создать заявку
   ↓
2. 🏬 Выбор склада
   ↓
3. 🛒 Выбор товаров и количества
   ↓
4. ✅ Продолжить
   ↓
5. 👤 Выбор клиента (inline-кнопки)  ← НОВОЕ!
   ↓
6. 🚚 Номер транспорта
   ↓
7. 📝 Комментарий
   ↓
8. ✅ Подтверждение
```

## Технические детали

### Изменения в коде:

1. **Новый шаг `select_client`** - добавлен после выбора товаров (при нажатии "✅ Продолжить") только для администраторов
2. **Inline-клавиатура** - список клиентов отображается в виде inline-кнопок прямо в сообщении
3. **Обработчик callback_query** - добавлен новый обработчик для обработки нажатий на inline-кнопки
4. **Загрузка списка клиентов** - используется функция `database.getAllClients()` для получения всех активных клиентов
5. **Сохранение данных клиента** - выбранный клиент сохраняется в `data.selectedClient`, его имя и телефон автоматически подставляются
6. **Правильная привязка заявки** - при сохранении в БД используется `telegram_id` выбранного клиента, а не администратора

### Структура inline-кнопок:

```javascript
const inlineKeyboard = clients.map(client => [{
  text: `${client.name || 'Без имени'} (${client.phone || 'Без телефона'})`,
  callback_data: `select_client_${client.telegram_id}`
}]);
```

### Обработка callback:

```javascript
bot.on('callback_query', async (ctx) => {

```

### Структура данных заявки:

```javascript
{
  step: 'select_client',           // Текущий шаг
  warehouse: 'Склад 1',            // Выбранный склад
  items: [                         // Добавленные товары
    { product: 'Товар 1', quantity: '100 шт' }
  ],
  clientsList: [...],              // Список всех клиентов
  selectedClient: {                // Выбранный клиент
    telegram_id: 123456789,
    name: 'Иван Иванов',
    phone: '+992900000000'
  },
  name: 'Иван Иванов',            // Имя из выбранного клиента
  phone: '+992900000000',          // Телефон из выбранного клиента
  transport: '1234 AB',
  comment: 'Комментарий'
}
```

## Преимущества:

✅ Администратор может создавать заявки от имени любого клиента  
✅ Не нужно вручную вводить имя и телефон клиента  
✅ Исключены ошибки при вводе контактных данных  
✅ Заявка правильно привязывается к клиенту в базе данных  
✅ Статистика и отчеты корректно отображают заявки клиентов  
✅ Выбор клиента происходит в логичном месте - после формирования списка товаров  
✅ Удобный интерфейс с inline-кнопками прямо в сообщении  
✅ После выбора кнопки исчезают, и сообщение обновляется с выбранным клиентом  

## Безопасность:

- Выбор клиента доступен только администраторам
- Обычные клиенты продолжают создавать заявки только от своего имени
- Проверка прав доступа выполняется на каждом шаге и в обработчике callback
