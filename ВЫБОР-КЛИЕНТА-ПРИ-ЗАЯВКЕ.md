# Выбор клиента при создании заявки администратором

## Описание изменений

Добавлена возможность для администратора искать и выбирать клиента из списка зарегистрированных клиентов при создании заявки на поставку товаров. Система использует поиск по имени или телефону клиента.

## Как это работает

### Для администратора:

1. Нажмите кнопку **"📦 Создать заявку"**
2. Выберите склад
3. Добавьте товары в заявку
4. Нажмите **"✅ Продолжить"**
5. Введите имя клиента или часть имени (минимум 2 символа)
   - Например: "Иван", "Ива", "900" (часть телефона)
6. Система покажет найденных клиентов в виде inline-кнопок (максимум 10)
7. Нажмите на кнопку с нужным клиентом
8. Система автоматически подставит имя и телефон выбранного клиента
9. Введите номер транспорта и комментарий
10. Подтвердите заявку

### Для обычных клиентов:

Процесс создания заявки остался без изменений - клиенты продолжают использовать кнопку **"🏬 Склад"** и заполняют заявку от своего имени.

## Примеры поиска:

- **"Иван"** - найдет всех клиентов с "Иван" в имени
- **"Ива"** - найдет "Иван", "Иванов", "Ивановна" и т.д.
- **"900"** - найдет клиентов с "900" в номере телефона
- **"+992"** - найдет клиентов с таджикскими номерами

## Технические детали

### Изменения в коде:

1. **Новый шаг `search_client`** - запрос поискового запроса от администратора
2. **Поиск по имени и телефону** - система ищет совпадения в обоих полях (регистронезависимый поиск)
3. **Ограничение результатов** - показывается максимум 10 клиентов для удобства
4. **Inline-клавиатура** - найденные клиенты отображаются в виде inline-кнопок
5. **Обработчик callback_query** - обработка нажатий на inline-кнопки
6. **Правильная привязка заявки** - при сохранении в БД используется `telegram_id` выбранного клиента

### Логика поиска:

```javascript
// Поиск клиентов по имени или телефону
const foundClients = allClients.filter(client => {
  const clientName = (client.name || '').toLowerCase();
  const clientPhone = (client.phone || '').toLowerCase();
  return clientName.includes(searchQuery) || clientPhone.includes(searchQuery);
});

// Ограничение до 10 результатов
const limitedClients = foundClients.slice(0, 10);
```

### Структура inline-кнопок:

```javascript
const inlineKeyboard = limitedClients.map(client => [{
  text: `${client.name || 'Без имени'} (${client.phone || 'Без телефона'})`,
  callback_data: `select_client_${client.telegram_id}`
}]);
```

### Обработка callback:

```javascript
bot.on('callback_query', async (ctx) => {
  if (callbackData.startsWith('select_client_')) {
    const clientTelegramId = callbackData.replace('select_client_', '');
    // Находим клиента по telegram_id (сравнение как строки)
    const selectedClient = data.clientsList.find(client => 
      String(client.telegram_id) === String(clientTelegramId)
    );
    // Сохраняем данные и переходим к следующему шагу
  }
});
```

### Структура данных заявки:

```javascript
{
  step: 'search_client',           // Шаг поиска клиента
  warehouse: 'Склад 1',            // Выбранный склад
  items: [                         // Добавленные товары
    { product: 'Товар 1', quantity: '100 шт' }
  ],
  clientsList: [...],              // Список найденных клиентов (до 10)
  selectedClient: {                // Выбранный клиент
    telegram_id: 123456789,
    name: 'Иван Иванов',
    phone: '+992900000000'
  },
  name: 'Иван Иванов',            // Имя из выбранного клиента
  phone: '+992900000000',          // Телефон из выбранного клиента
  transport: '1234 AB',
  comment: 'Комментарий'
}
```

## Преимущества:

✅ Быстрый поиск клиента по имени или телефону  
✅ Не нужно прокручивать длинный список всех клиентов  
✅ Поддержка частичного совпадения (можно ввести несколько букв)  
✅ Поиск работает и по имени, и по телефону  
✅ Ограничение результатов до 10 для удобства  
✅ Администратор может создавать заявки от имени любого клиента  
✅ Исключены ошибки при вводе контактных данных  
✅ Заявка правильно привязывается к клиенту в базе данных  
✅ Удобный интерфейс с inline-кнопками прямо в сообщении  
✅ После выбора кнопки исчезают, и сообщение обновляется  

## Безопасность:

- Поиск и выбор клиента доступен только администраторам
- Обычные клиенты продолжают создавать заявки только от своего имени
- Проверка прав доступа выполняется на каждом шаге и в обработчике callback
- Минимальная длина поискового запроса - 2 символа

## Обработка ошибок:

- Если введено менее 2 символов - запрос на ввод большего количества
- Если клиенты не найдены - предложение попробовать другой запрос
- Если в системе нет клиентов - уведомление о необходимости добавить клиентов
