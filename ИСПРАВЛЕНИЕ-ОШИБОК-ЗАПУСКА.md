# 🔧 ИСПРАВЛЕНИЕ ОШИБОК ЗАПУСКА БОТА

## ❌ ПРОБЛЕМЫ

1. **TypeError: Cannot set properties of undefined**
   - Ошибка в `fix-order-bot-soft-delete.js`
   - Пытается добавить методы к `prototype` вместо экземпляра

2. **SQLITE_CANTOPEN: unable to open database file**
   - Нет прав на создание файла `orders.db`
   - Неправильный путь в контейнере

## ✅ РЕШЕНИЕ

### Шаг 1: Исправить soft-delete

Заменить в `bot.js` строку:
```javascript
const orderBotFixes = require('./fix-order-bot-soft-delete');
```

На:
```javascript
const orderBotFixes = require('./fix-order-bot-soft-delete-v2');
```

### Шаг 2: Исправить путь к базе данных

Добавить в начало `bot.js` (перед require('./database')):
```javascript
// Исправление пути к БД для контейнера
require('./fix-database-path');
```

### Шаг 3: Создать директорию для данных

В `Dockerfile` или при запуске контейнера:
```dockerfile
# Создаем директорию для данных
RUN mkdir -p /app/data && chmod 777 /app/data

# Или используем volume
VOLUME /app/data
```

### Шаг 4: Установить переменную окружения

В `.env` или Railway:
```env
DB_PATH=/app/data/orders.db
```

## 🚀 БЫСТРОЕ ИСПРАВЛЕНИЕ

Если нет доступа к Dockerfile, просто измените `bot.js`:

```javascript
// В НАЧАЛЕ ФАЙЛА (строка 1-5)
require('dotenv').config();

// ДОБАВИТЬ ЭТИ СТРОКИ:
const fs = require('fs');
const dataDir = '/tmp';
if (!fs.existsSync(dataDir)) {
    fs.mkdirSync(dataDir, { recursive: true });
}
process.env.DB_PATH = '/tmp/orders.db';

// Остальной код...
const { Telegraf, Scenes, session } = require('telegraf');
const database = require('./database');
```

## 📝 АЛЬТЕРНАТИВА: Использовать PostgreSQL

Для production лучше использовать PostgreSQL вместо SQLite:

1. Установить `pg`:
```bash
npm install pg
```

2. Создать `database-postgres.js`
3. Изменить `require('./database')` на `require('./database-postgres')`

## 🔍 ОТЛАДКА

Если ошибки продолжаются:

1. Проверить права на директорию:
```bash
ls -la /app/data
```

2. Проверить переменные окружения:
```bash
echo $DB_PATH
```

3. Проверить логи:
```bash
npm start 2>&1 | tee bot.log
```

## 📞 ПОДДЕРЖКА

Если проблемы остаются, проверьте:
- Права доступа к файловой системе
- Наличие директории /app/data
- Переменные окружения
- Версию Node.js (должна быть >= 16)
