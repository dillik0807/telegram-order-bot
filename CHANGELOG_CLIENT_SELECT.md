# Изменения: Выбор клиента из списка при создании заявки

## Описание
Добавлена функциональность выбора клиента из списка с помощью inline-клавиатуры при создании заявки администратором. Администратор сначала заполняет заявку (склад, товары), а затем выбирает клиента из списка.

## Что изменилось

### 1. Новый процесс создания заявки для администратора
- Администратор нажимает "📦 Создать заявку"
- Выбирает склад
- Добавляет товары
- Нажимает "✅ Продолжить"
- **НОВОЕ:** Видит список клиентов в виде inline-кнопок (первые 10)
- Выбирает клиента
- Вводит номер транспорта
- Добавляет комментарий
- Подтверждает заявку

### 2. Поиск клиента в процессе создания заявки
- Команда `/findclient Имя` работает в контексте создания заявки
- Поиск по имени или телефону (регистронезависимый)
- Результаты отображаются с правильным callback для текущего контекста

### 3. Обработчики callback
- `selectclient_{telegram_id}` - выбор клиента в процессе создания заявки
- `client_{telegram_id}` - выбор клиента при создании заявки с начала (старый способ)
- `cancel_order` - отмена создания заявки
- `show_more_info` - информационная кнопка о количестве клиентов

### 4. Изменения в файлах

#### bot.js
- Изменена логика обработки "✅ Продолжить" - для администратора показывается список клиентов
- Команда `/findclient` теперь учитывает контекст (создание заявки или просто поиск)
- Обработчик "📦 Создать заявку" возвращен к стандартному поведению (сразу выбор склада)

#### admin.js
- Добавлен обработчик `bot.action(/selectclient_(\d+)/)` для выбора клиента в процессе заявки
- Обработчик `bot.action(/client_(\d+)/)` оставлен для совместимости
- Обработчик `bot.action('cancel_order')` для отмены создания заявки

## Как использовать

### Для администратора:

1. **Создание заявки с выбором клиента:**
   ```
   Нажать "📦 Создать заявку"
   → Выбрать склад
   → Добавить товары (можно несколько)
   → Нажать "✅ Продолжить"
   → Выбрать клиента из списка (inline-кнопки)
   → Ввести номер транспорта
   → Добавить комментарий (или пропустить)
   → Подтвердить заявку
   ```

2. **Поиск клиента в процессе создания заявки:**
   ```
   После нажатия "✅ Продолжить" и появления списка клиентов:
   /findclient Алишер
   → Выбрать нужного клиента из результатов
   → Продолжить создание заявки
   ```

3. **Отмена создания заявки:**
   ```
   Нажать кнопку "❌ Отмена" в списке клиентов
   ```

## Преимущества

✅ Администратор сначала видит всю заявку (склад, товары)
✅ Затем выбирает клиента из списка без ручного ввода
✅ Исключены ошибки при вводе имени и телефона
✅ Удобный поиск по большому списку клиентов
✅ Inline-клавиатура не занимает место в чате
✅ Данные клиента автоматически подставляются в заявку

## Совместимость

- Для обычных клиентов процесс создания заявки не изменился
- Клиенты по-прежнему используют кнопку "🏬 Склад" и их данные подставляются автоматически
- Изменения касаются только администраторов

## Технические детали

- Используется Map `orderData` для хранения состояния создания заявки
- Шаг `select_client_for_order` добавлен для администраторов после выбора товаров
- Inline-клавиатура ограничена 10 клиентами для удобства
- Callback data формата: `selectclient_{telegram_id}` для выбора в процессе заявки
- Поиск работает по полям `name` и `phone` (case-insensitive)
- Команда `/findclient` адаптируется к контексту использования
