# Использование нескольких Green-API аккаунтов для разных складов

## Решение проблемы квоты

Если у вас закончилась квота на одном аккаунте Green-API, вы можете создать несколько аккаунтов и привязать каждый склад к своему аккаунту.

## Как это работает

Каждый склад может иметь свой собственный Green-API инстанс:
- Склад "ОБЛ База" → Green-API аккаунт #1
- Склад "ЧБалхи" → Green-API аккаунт #2
- Склад "Центральный" → Green-API аккаунт #3

## Пошаговая настройка

### Шаг 1: Создайте новые аккаунты Green-API

1. Перейдите на https://green-api.com
2. Зарегистрируйте новый аккаунт (используйте другой email)
3. Создайте новый инстанс
4. Отсканируйте QR-код для авторизации WhatsApp
5. Скопируйте `Instance ID` и `Token`

Повторите для каждого склада.

### Шаг 2: Привяжите Green-API инстанс к складу

```
/setgreenapi Название_склада | Instance_ID | Token
```

**Примеры:**

```bash
# Склад "ОБЛ База" → Аккаунт #1
/setgreenapi ОБЛ База | 1234567890 | abc123def456ghi789

# Склад "ЧБалхи" → Аккаунт #2
/setgreenapi ЧБалхи | 9876543210 | xyz789uvw012rst345

# Склад "Центральный" → Аккаунт #3
/setgreenapi Центральный | 5555555555 | qwe456rty789uio012
```

### Шаг 3: Настройте номер или группу для склада

После привязки Green-API настройте, куда отправлять сообщения:

**Вариант А: Личный номер**
```
/setwhatsappphone ОБЛ База | 992935020807
/setwhatsappphone ЧБалхи | 992901234567
```

**Вариант Б: Группа WhatsApp**
```
/setwhatsapp ОБЛ База | 120363142919317671@g.us
/setwhatsapp ЧБалхи | 120363419535622239@g.us
```

### Шаг 4: Проверьте настройки

```
/checkwarehouses
```

Должно показать:
```
📋 Список складов:

1. ОБЛ База (ID: 1)
   📱 Группа: ❌ | Номер: ✅ | Green-API: ✅
   └─ Номер: +992935020807
   └─ Instance: 1234567890

2. ЧБалхи (ID: 2)
   📱 Группа: ❌ | Номер: ✅ | Green-API: ✅
   └─ Номер: +992901234567
   └─ Instance: 9876543210
```

## Приоритет использования Green-API

Система использует следующий приоритет:

```
1️⃣ Green-API инстанс склада (если настроен)
   ↓
2️⃣ Общий Green-API из .env файла
```

Если у склада настроен свой инстанс, он будет использоваться. Если нет - используется общий из `.env`.

## Преимущества

✅ Каждый склад имеет свою квоту сообщений  
✅ Нет ограничений на количество корреспондентов  
✅ Можно использовать разные номера WhatsApp  
✅ Независимые аккаунты - если один заблокирован, другие работают  
✅ Легко масштабировать - просто добавляйте новые аккаунты  

## Пример полной настройки

### Создайте 3 аккаунта Green-API:

**Аккаунт #1:**
- Email: warehouse1@example.com
- Instance ID: 1111111111
- Token: token1token1token1

**Аккаунт #2:**
- Email: warehouse2@example.com
- Instance ID: 2222222222
- Token: token2token2token2

**Аккаунт #3:**
- Email: warehouse3@example.com
- Instance ID: 3333333333
- Token: token3token3token3

### Привяжите к складам:

```bash
# Склад 1
/setgreenapi ОБЛ База | 1111111111 | token1token1token1
/setwhatsappphone ОБЛ База | 992935020807

# Склад 2
/setgreenapi ЧБалхи | 2222222222 | token2token2token2
/setwhatsappphone ЧБалхи | 992901234567

# Склад 3
/setgreenapi Центральный | 3333333333 | token3token3token3
/setwhatsappphone Центральный | 992909999999
```

### Проверьте:

```
/checkwarehouses
```

## Структура базы данных

Таблица `warehouses` теперь содержит:

```sql
CREATE TABLE warehouses (
  id SERIAL PRIMARY KEY,
  name TEXT NOT NULL UNIQUE,
  whatsapp_group_id TEXT,              -- ID группы WhatsApp
  whatsapp_phone TEXT,                 -- Личный номер WhatsApp
  green_api_instance_id TEXT,          -- Instance ID Green-API (НОВОЕ!)
  green_api_token TEXT,                -- Token Green-API (НОВОЕ!)
  is_active INTEGER DEFAULT 1,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## Обновление существующих складов

Если у вас уже есть склады, просто добавьте к ним Green-API:

```bash
# Проверьте текущие склады
/checkwarehouses

# Добавьте Green-API к существующим складам
/setgreenapi Существующий_склад | Instance_ID | Token
```

Все остальные настройки (номер, группа) сохранятся.

## Отвязка Green-API от склада

Если нужно вернуться к использованию общего Green-API:

```bash
/setgreenapi Название_склада | | 
```

(пустые значения после разделителей)

Или обновите запись в базе данных:

```sql
UPDATE warehouses 
SET green_api_instance_id = NULL, green_api_token = NULL 
WHERE name = 'Название_склада';
```

## Тестирование

После настройки создайте тестовую заявку для каждого склада и проверьте:
1. Сообщение приходит на правильный номер
2. В логах указан правильный Instance ID
3. Нет ошибок квоты

## Стоимость

Каждый аккаунт Green-API имеет свой тариф. Рекомендуется:
- Для активных складов: Business тариф (без ограничений)
- Для малоактивных складов: Developer тариф (с ограничениями)

## Автоматическое создание колонок

При перезапуске бота колонки `green_api_instance_id` и `green_api_token` будут автоматически добавлены в таблицу `warehouses`, если их нет.

```bash
npm start
```

В логах вы увидите:
```
✅ Колонки green_api_instance_id и green_api_token добавлены
```

Готово! Теперь каждый склад может использовать свой Green-API аккаунт!
