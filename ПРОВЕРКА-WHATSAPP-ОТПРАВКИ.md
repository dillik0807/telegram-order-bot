# Проверка отправки заявок в WhatsApp

## Текущая реализация

### Логика отправки заявок:

1. **Сохранение в БД** - заявка сохраняется в PostgreSQL
2. **Отправка в Telegram** - заявка отправляется в Telegram группу (если настроена)
3. **Умная маршрутизация WhatsApp** - заявка отправляется в WhatsApp по следующей логике:

```
┌─────────────────────────────────────┐
│ Есть WhatsApp группа для склада?    │
└──────────┬──────────────────────────┘
           │
    ┌──────┴──────┐
    │ ДА          │ НЕТ
    ▼             ▼
┌───────────┐  ┌──────────────────────┐
│ Отправка  │  │ Есть общая группа?   │
│ в группу  │  └──────┬───────────────┘
│ склада    │         │
└───────────┘    ┌────┴────┐
                 │ ДА      │ НЕТ
                 ▼         ▼
            ┌─────────┐  ┌──────────────┐
            │ Отправка│  │ Есть личный  │
            │ в общую │  │ получатель?  │
            │ группу  │  └──────┬───────┘
            └─────────┘         │
                           ┌────┴────┐
                           │ ДА      │ НЕТ
                           ▼         ▼
                      ┌─────────┐  ┌──────────┐
                      │ Отправка│  │ Не       │
                      │ личному │  │ отправка │
                      └─────────┘  └──────────┘
```

## Модуль WhatsApp (whatsapp.js)

### Функция `sendMessage(message)`
Отправляет сообщение личному получателю через Green-API.

**Параметры:**
- `message` - текст сообщения

**Переменные окружения:**
- `GREEN_API_INSTANCE_ID` - ID инстанса Green-API
- `GREEN_API_TOKEN` - токен Green-API
- `WHATSAPP_RECIPIENT` - номер получателя (например: 992900000000)

**Формат chatId:** `{номер}@c.us` (для личного чата)

### Функция `sendToGroup(message, groupId)`
Отправляет сообщение в WhatsApp группу через Green-API.

**Параметры:**
- `message` - текст сообщения
- `groupId` - ID группы WhatsApp

**Переменные окружения:**
- `GREEN_API_INSTANCE_ID` - ID инстанса Green-API
- `GREEN_API_TOKEN` - токен Green-API

**Формат groupId:** `120363XXXXXXXXXX@g.us` (для группы)

## Проверка работы

### 1. Проверьте переменные окружения (.env файл):

```env
# Green-API настройки
GREEN_API_INSTANCE_ID=ваш_instance_id
GREEN_API_TOKEN=ваш_токен

# WhatsApp получатель (для личных сообщений)
WHATSAPP_RECIPIENT=992900000000

# WhatsApp группа (общая, если нет привязки к складу)
WHATSAPP_GROUP_ID=120363XXXXXXXXXX@g.us
```

### 2. Проверьте привязку WhatsApp групп к складам:

В базе данных PostgreSQL в таблице `warehouses` должна быть колонка `whatsapp_group_id`:

```sql
SELECT name, whatsapp_group_id FROM warehouses;
```

Если колонки нет, запустите миграцию:
```bash
node telegram-order-bot/add-whatsapp-column.js
```

### 4. Возможные проблемы:

#### ❌ "Green-API не настроен"
**Причина:** Отсутствуют `GREEN_API_INSTANCE_ID` или `GREEN_API_TOKEN` в .env файле  
**Решение:** Добавьте эти переменные в .env файл

#### ❌ "ID группы WhatsApp не указан"
**Причина:** Не указан `groupId` при вызове `sendToGroup()`  
**Решение:** Проверьте привязку группы к складу в БД или укажите `WHATSAPP_GROUP_ID` в .env

#### ❌ "Ошибка отправки в WhatsApp группу"
**Причина:** Неверный формат `groupId` или проблемы с Green-API  
**Решение:** 
- Проверьте формат groupId (должен быть `120363XXXXXXXXXX@g.us`)
- Проверьте, что бот добавлен в группу WhatsApp
- Проверьте статус Green-API инстанса

## Важные замечания

### Для администратора, создающего заявку для клиента:

✅ **Правильно:** В сообщении WhatsApp будет указано имя и телефон выбранного клиента  
✅ **Правильно:** Заявка сохраняется в БД с привязкой к клиенту (не к администратору)  
✅ **Правильно:** Статистика будет корректно отображать заявки клиентов  

### Формат данных в сообщении:

```javascript
const orderMessage = formatOrder({
  name: data.name,              // Имя клиента (из выбранного или текущего)
  phone: data.phone,            // Телефон клиента
  warehouse: data.warehouse,    // Выбранный склад
  items: data.items,            // Массив товаров
  transport: data.transport,    // Номер транспорта
  comment: data.comment         // Комментарий (опционально)
});
```

## Проверка после внедрения выбора клиента

1. ✅ В WhatsApp сообщении отображается имя выбранного клиента, а не администратора
