📋 ОПИСАНИЕ:
───────────────────────────────────────────────────────────────
Добавлена новая функция для администраторов - возможность 
выбирать клиента из существующего списка при создании заявки
вместо ручного ввода имени и номера телефона.

🎯 ДЛЯ КОГО:
───────────────────────────────────────────────────────────────
✅ Администраторы - видят выбор метода заполнения
❌ Обычные пользователи - работают как раньше

📱 КАК РАБОТАЕТ:
───────────────────────────────────────────────────────────────

🔧 ТЕХНИЧЕСКИЕ ДЕТАЛИ:
───────────────────────────────────────────────────────────────

Новые шаги в процессе создания заявки:

• select_client_method
  Выбор метода заполнения данных

• select_client
  Выбор конкретного клиента из списка

Формат отображения клиента:
  "Имя (Телефон)"
  Пример: "Алишер Иванов (+992901234567)"

Фильтрация клиентов:
  Показываются только клиенты с заполненными
  именем и телефоном

Сохранение данных:
  data.name = selectedClient.name
  data.phone = selectedClient.phone
  data.selectedClientId = selectedClient.telegram_id

❓ ЧАСТЫЕ ВОПРОСЫ:
───────────────────────────────────────────────────────────────

Q: Я администратор, но не вижу выбор метода?
A: Проверьте, что ваш Telegram ID добавлен в список
   ADMINS в файле admin.js

Q: Список клиентов пустой?
A: Убедитесь, что в базе есть клиенты с заполненными
   именем и телефоном. Используйте /addclient для
   добавления клиентов.

Q: Можно ли добавить нового клиента во время создания заявки?
A: Да! Выберите "✍️ Ввести данные вручную" и введите
   данные нового клиента.

Q: Обычные пользователи видят эту функцию?
A: Нет, только администраторы. Обычные пользователи
   работают как раньше.

Q: Нужно ли обновлять базу данных?
A: Нет, изменения не требуют миграции базы данных.
